# 🐳 Docker Configuration Files

This folder now contains all necessary files to containerize and deploy the Strapi CMS on Hostinger VPS.

## 📁 New Files Created

### Core Docker Files

1. **Dockerfile**
   - Multi-stage build optimized for Strapi
   - Uses Node.js 20 Alpine Linux for minimal size
   - Includes security best practices (non-root user, tini init)
   - Health checks enabled

2. **.dockerignore**
   - Excludes unnecessary files from Docker build context
   - Reduces build time and image size

3. **docker-compose.yml**
   - Full stack with Strapi + Nginx reverse proxy
   - Recommended for production with SSL/TLS
   - Includes health checks and volume management

4. **docker-compose.prod.yml**
   - Simplified production setup (Strapi only)
   - For use with external Nginx/reverse proxy
   - Includes resource limits and security hardening

### Configuration Files

5. **nginx.conf**
   - Nginx reverse proxy configuration
   - Includes HTTP→HTTPS redirect
   - SSL/TLS support for Let's Encrypt certificates
   - Compression, caching, and security headers

6. **.env.example**
   - Updated with Docker-specific variables
   - Security keys configuration
   - Database and server settings

### Deployment & Documentation

7. **deploy.sh**
   - Automated deployment script for Hostinger VPS
   - Includes backup creation before deployment
   - Health checks and error handling
   - One-command deployment

8. **DOCKER_DEPLOYMENT.md**
   - Complete deployment guide for Hostinger
   - Step-by-step instructions
   - SSL/TLS configuration with Let's Encrypt
   - Troubleshooting guide
   - Security checklist

## 🚀 Quick Start

### Local Development

```bash
# Build and run locally
docker-compose up --build

# Access at http://localhost:1337/admin
```

### Hostinger VPS Deployment

```bash
# 1. Configure environment
cp .env.example .env
nano .env  # Edit with your values

# 2. Make deploy script executable
chmod +x deploy.sh

# 3. Run deployment
./deploy.sh

# 4. Verify
docker-compose ps
curl http://localhost:1337/admin
```

## 🔧 Configuration Options

### Using Different Compose Configs

```bash
# Full stack with Nginx
docker-compose -f docker-compose.yml up -d

# Lightweight production (external reverse proxy)
docker-compose -f docker-compose.prod.yml up -d
```

### Environment Variables

Key variables to configure in `.env`:

| Variable | Purpose | Example |
|----------|---------|---------|
| `APP_KEYS` | Security keys (comma-separated) | Generated by deployment |
| `ADMIN_JWT_SECRET` | Admin authentication | Generated by deployment |
| `DATABASE_FILENAME` | SQLite DB filename | data.db |
| `STRAPI_PORT` | Strapi container port | 1337 |
| `NGINX_PORT` | Nginx HTTP port | 80 |
| `NGINX_SSL_PORT` | Nginx HTTPS port | 443 |

## 📝 Important Notes

### Before Deployment

1. **Generate Security Keys**
   ```bash
   node -e "console.log(require('crypto').randomBytes(16).toString('base64'))"
   ```
   Run this multiple times for `APP_KEYS`

2. **Data Persistence**
   - Database and uploads stored in volumes
   - Back up `data/` directory regularly
   - Use `deploy.sh` which creates automatic backups

3. **SSL/TLS Certificate**
   - Use Let's Encrypt for free SSL
   - Automatic renewal available (see DOCKER_DEPLOYMENT.md)

### Monitoring

```bash
# View real-time logs
docker-compose logs -f strapi

# Check container health
docker ps

# View resource usage
docker stats
```
